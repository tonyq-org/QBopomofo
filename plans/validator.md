# QBopomofo 輸入行為驗證規則

本文件記錄所有經過手動驗證的輸入行為預期，作為未來自動化測試和迴歸驗證的依據。

每條規則包含：操作步驟、預期結果、實際驗證日期。

---

## 一、基本中文輸入

### V-001: 注音組字基本流程
- **操作**: 打 `hk4` (ㄘㄜˋ)
- **預期**: 組字區顯示「測」
- **驗證**: 2026-03-30

### V-002: 多字組字
- **操作**: 打 `hk4g4` (ㄘㄜˋ ㄕˋ)
- **預期**: 組字區顯示「測試」
- **驗證**: 2026-03-30

### V-003: Enter 送出組字
- **操作**: 打 `hk4g4` → Enter
- **預期**: 輸出文字「測試」，組字區清空
- **驗證**: 2026-03-30

### V-004: Backspace 刪除注音
- **操作**: 打 `hk4` → Backspace
- **預期**: 組字區清空（刪除整個音節或最後一個注音符號）
- **驗證**: 2026-03-30

### V-005: Esc 清除組字區
- **操作**: 打 `hk4g4` → Esc
- **預期**: 組字區清空，已輸出的文字不受影響
- **驗證**: 2026-03-30

## 二、Shift SmartToggle 行為

### V-010: Shift 短按切換中英
- **操作**: （中文模式）短按 Shift（按下即放開，不打字）
- **預期**: 切換到英文模式，狀態指示從「中」→「英」
- **驗證**: 2026-03-30

### V-011: Shift 再短按切回中文
- **操作**: （英文模式）短按 Shift
- **預期**: 切回中文模式，狀態指示從「英」→「中」
- **驗證**: 2026-03-30

### V-012: Shift 長按暫時英文
- **操作**: （中文模式）按住 Shift 不放 → 打 `ABC` → 放開 Shift
- **預期**: 打字時為暫時英文模式，放開後回到中文模式
- **驗證**: 2026-03-30

---

## 三、英文直接輸出（無中文組字時）

### V-020: 切換英文模式直接打字
- **操作**: （組字區空）Shift 切換到英文 → 打 `abc`
- **預期**: 輸出文字直接顯示 `abc`（不進入組字區）
- **驗證**: 2026-03-30

### V-021: 按住 Shift 暫時英文（組字區空）
- **操作**: （組字區空）按住 Shift → 打 `ABC`
- **預期**: 輸出文字直接顯示 `ABC`（不進入組字區）
- **驗證**: 2026-03-30

## 四、中英混排組字

### V-030: 中文後接英文進入混排
- **操作**: 打 `hk4g4` (測試) → Shift → 打 `123` → Shift → 打 `hk4` (測) → Enter
- **預期組字區**: `測試123測`（英文夾在中文中間）
- **預期輸出**: `測試123測`（按插入順序）
- **驗證**: 2026-03-30

### V-031: 按住 Shift 混排（字母）
- **操作**: 打 `hk4` (測) → 按住 Shift → 打 `A` → 放開 Shift
- **預期組字區**: `測A`（中文在前，英文在後）
- **驗證**: 2026-03-30 (**BUG 發現：顯示為 `A測`，已修正**)

### V-032: 混排 Esc 清除全部
- **操作**: 打 `hk4` → Shift → 打 `ABC` → Esc
- **預期**: 組字區完全清空（中文 + 英文都清除），回到中文模式
- **驗證**: 2026-03-30

### V-033: 混排 Backspace 刪英文
- **操作**: 打 `hk4` (測) → 按住 Shift → 打 `AAA` → 放開 Shift → Backspace ×3
- **預期**: 每次 Backspace 刪除一個英文字母，最終組字區只剩「測」
- **驗證**: 2026-03-30 (**BUG 發現：英文存入 segment 後 Backspace 無法刪除，已修正**)

### V-034: 多段混排正確排序
- **操作**: 打 `hk4g4m/4` (測試用) → Shift → 打 `123` → Shift → 打 `hk4` (測) → Enter
- **預期輸出**: `測試用123測`（非 `123測試用測` 或 `測試用測123`）
- **驗證**: 2026-03-30 (**BUG 發現：排序錯誤，已修正**)

### V-035: 按住 Shift + 標點鍵 → 中文標點（非英文）
- **操作**: 打 `hk4g4` (測試) → 按住 Shift → 打 `,` (鍵盤送出 `<`) → 放開 Shift
- **預期**: 組字區顯示「測試，」（中文逗號，非英文 `<`），且放開 Shift 後維持中文模式
- **說明**: 按住 Shift 時，只有字母 (a-z) 走 temp English 路徑，標點符號 fall through 給 chewing 引擎產生中文標點。非字母按鍵仍標記 `shift_typed_while_held`，防止 Shift 放開時誤切英文模式
- **驗證**: 2026-03-30 (**BUG 發現：(1) 顯示為 `測試<`；(2) 放開 Shift 後誤切英文模式，已修正**)

### V-036: 按住 Shift + 標點後繼續打中文
- **操作**: 打 `hk4g4m/4` (測試用) → 按住 Shift → 打 `,` → 放開 Shift → 打 `au4` (這)
- **預期**: 組字區顯示「測試用，這」，持續中文模式
- **驗證**: 2026-03-30

### V-037: 組字區超過 20 字自動 flush
- **操作**: 切換英文模式 → 連續打超過 20 個英文字母
- **預期**: 超過 20 字時自動 commit，組字區清空
- **說明**: 中文+英文合計組字區顯示超過 20 字即觸發 auto-flush commitAll。純英文時完全 flush，混合時整段 flush
- **驗證**: 2026-03-30

- **驗證方式**: Code review + profiling
