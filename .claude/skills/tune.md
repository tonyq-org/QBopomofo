# /tune — 詞庫調校 Skill

## 描述
協助使用者新增、調整、或移除自訂詞彙，確保選字準確度。

## 使用方式
```
/tune 再試試 應該優先於 在試試
/tune 新增「人工智慧」
/tune 「心的」和「新的」哪個應該優先
```

## 執行流程

收到使用者的調校需求後，依序執行以下步驟：

### 1. 分析需求
- 理解使用者想要的效果（哪個詞應該優先出現）
- 確認注音拼法（用注音符號表示）

### 2. 衝突檢查
搜尋現有詞庫，檢查是否有衝突：
- 讀取 `data-provider/chewing-data/tsi.csv` 查詢上游詞頻
- 讀取 `data-provider/custom-data/phrases.csv` 查詢已有的自訂調整
- 用 Grep 搜尋相關的同音詞，列出它們的現有詞頻
- 分析：調整後是否會影響其他常用詞的排序？

### 3. 提出建議
向使用者報告分析結果，包含：
- 相關同音詞的現有詞頻表格
- 建議的新詞頻值及理由
- 可能受影響的其他詞彙
- 請使用者確認後再寫入

### 4. 寫入詞庫
使用者確認後：
- 編輯 `data-provider/custom-data/phrases.csv`
- 在檔案末尾新增條目，格式：`詞,詞頻,注音1 注音2 ...`
- 每行加上 `# 備註` 說明調整原因

### 5. 重建 & 測試
```bash
source "$HOME/.cargo/env" && ./data-provider/build.sh
```
建議使用者用 TestApp 測試：
```bash
cd mac/TestApp && swift run
```

## 詞頻參考值

| 等級 | 範圍 | 範例 |
|------|------|------|
| 超高頻 | 50000+ | 的、是、我、不、在 |
| 高頻 | 5000-50000 | 已經、可以、因為、這樣 |
| 中頻 | 500-5000 | 嘗試、調整、新增 |
| 低頻 | 0-500 | 罕用詞 |

## 重要原則

- **不要直接修改 `chewing-data/tsi.csv`** — 那是上游資料，只改 `custom-data/phrases.csv`
- 同音詞的選字優先級由詞頻決定，詞頻高的優先
- 如果兩個詞同音且詞頻差距太小（< 100），建議拉開差距以確保穩定性
- 加詞時考慮是否有多音字情況（如「再」ㄗㄞˋ）
- 每次修改都要附上理由註解
